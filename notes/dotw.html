1. Introduction

The problem is stated as following:

Write a program that takes a date as input and prints the day of the week that date falls on. We use the Gregorian calendar for this calculation.

Inputs:

Accept 3 integers as month, day and year. Use 1 for January, 2 for February, …

Outputs:

0 for Sunday, 1 for Monday, …

2. Simple methods: Julian and Rata Die

The main idea of this method (and all related algorithms) is that if we know the day of a specific date, we can infer from it. For example, if we know today is Monday; it means the next 7, 14, 21 days are also Monday. Let say we know that Jan 1st, 1 AD is Monday, given any date, we need to compute how many days (delta) has past since Jan 1st 1. The quantity then is modulo by 7, resulting the day of the input date. This simple method is called Rata Die. Rate Die number is precisely the number of passing days since Jan 1st, 1 AD.

{src/dotw.cr 2} 

def rata_die_dotw(day : Int, month : Int, year : Int) : Int 
  rata_die_date(day, month, year) % 7 
end

Added to in section 3

Instead of counting number of passing days, Rata Die is computed based on the Julian date. The computation is straightforward.

{src/dotw.cr 2} += 

def rata_die_date(day : Int, month : Int, year : Int) : Int
  (julian_day(day, month, year) - 1721424.5).floor.to_i
end

Added to in section 3

The Julian day is computed as following. This algorithm is developed by Henry F. Fliegel and Thomas C. Van Flandern Fliegel, H. F., & Van Flandern, T. C. (1968). Letters to the editor: a machine algorithm for processing calendar dates. Communications of the ACM, 11(10), 657.

. The original code (in Fortran) is much more compact than the implementation below:

{src/dotw.cr 2} += 

def julian_day(day : Int, month : Int, year : Int) : Int 
  month0 = (month - 14). tdiv 12
  result = (1461 * (year + 4800 + month0)).tdiv 4
  result += (367 * (month - 2 - 12 * month0)).tdiv 12
  result -= (3 * (year + 4900 + month0).tdiv 100).tdiv 4
  result += day - 32075
end

Added to in section 3

In Crystal, the division operator returns the result as a floating point number. While Crystal also supports the integer division // Digging deeper into this operator, I noticed that different programming languages implement integer division differently. See: Division and Modulus for Computer Scientists and Why Python’s Integer Division Floors

, it simply removes the fraction part of the result. What we need for this algorithm to work is the truncated division which returns the result toward 0. In other words, it rounds down (removing the fraction part) if the result is positive and round up otherwise. Fortunately, Crystal provides the tdiv that works exactly what we need.

Added to in section 2

After a common year, the day of the given date and month moves the the next day In case of leap years, it moves 2 days (for those start from March 1st) due to the additional day (Feb 29th).

The Tøndering method reformats the month starting from March instead of Jan.  Hence March would becomes 1, April is 2, so on and so far. Jan becomes 11 and Feb is 12. So if the input in Jan or Feb, then we need to start counting from the previous year.

The only tricky part left is (31 * month0)/ 12. It actually encodesZeller proposed using an arithmetic expression instead of a lookup table.

 the month offsets. We know that Jan 1st, 1 AD is Monday. Since Jan has 31 days (= 3 modulo 7), Feb 1st would be 3 days later, namely Thursday ((1 + 3) % 7 = 4). These 3 days in the calculation is in fact the month offsets: given an abitrary date and month, how many days we need to adjust from the day of that exact date in Jan.

{Tøndering method 3} 
month_offset = (14 - month) // 12
month0 = month + 12 * month_offset - 2 
delta_year = year - month_offset
delta_year = delta_year + delta_year // 4 - delta_year // 100 + delta_year // 400
(day + delta_year + (31 * month0) // 12) % 7

4. Testing

Let test this out

{spec/dotw_spec.cr 4} 
require "../src/dotw"
require "spec"

describe "day of the week methods" do 
  expects = [
    { {14, 2, 2000}, 1},
    { {2, 8, 1953}, 0},
    { {13, 4, 2022}, 3}
  ]

  it "return corrects result" do 
    expects.each do |x|
      tondering_dotw(*x[0]).should eq x[1]
      rata_die_dotw(*x[0]).should eq x[1]
    end
  end

  it "julian day" do 
    julian_day(1, 1, 2000). should eq 2451545
    julian_day(13, 4, 2022). should eq 2459683
  end
end
